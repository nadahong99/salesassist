# SalesAssist – 소싱/마진 계산기

네이버·쿠팡 소싱 상품의 마진을 계산하고 엑셀로 내보낼 수 있는 웹 앱입니다.  
**PC에서만 쓸 수도 있고, 클라우드에 올려서 핸드폰·아이패드 등 어디서나 쓸 수도 있습니다.**

---

## ✨ 주요 기능

| 기능 | 설명 |
|------|------|
| ⚙️ 비용 설정 | 수수료(%), 배송비, 기타비용, 목표 마진율, **부가세율(%)** 설정 |
| 🧮 빠른 마진 계산기 | 저장 없이 매입가·판매가를 입력하면 즉시 마진 계산 |
| 🔗 키워드·트렌드 바로가기 | 네이버 데이터랩, 쇼핑인사이트, 스마트스토어, 쿠팡Wing, 알리익스프레스, **도매꾹·사입삼촌·오너클랜(국내 도매)** 등 한 번에 이동 |
| ➕ 상품 관리 | 상품 추가·수정·삭제 — **소싱처 종류(해외/국내), 소싱처 이름, 소싱 URL, 최저가/도매가, 최소주문수량, 관부가세 유무** 포함 |
| 📊 마진 색상 표시 | 목표 달성(초록), 절반 이상(노랑), 미달(빨강) |
| 🔄 반품 체크리스트 | 상품별 반품 발생 여부 등록 및 후속 조치(환불·고객연락·재고반영·배송비·**세금계산서**) 체크리스트 |
| 📥 엑셀 내보내기 | 네이버·쿠팡별 마진 분석 엑셀 다운로드 (소싱처 종류·URL·도매가·관부가세·부가세·반품·세금계산서 현황 포함) |

## 🛒 국내·해외 소싱 관리 방법

### 소싱처 종류 구분

SalesAssist는 상품 등록 시 **소싱처 종류(해외/국내)** 를 선택할 수 있습니다.

| 구분 | 대표 플랫폼 | 특징 |
|------|------------|------|
| 🌏 **해외** | 타오바오, 알리익스프레스, 알리바바, 1688 | 관부가세 발생 가능, 배송일 길 수 있음 |
| 🇰🇷 **국내** | 도매꾹, 사입삼촌, 오너클랜, 도매토피아 | 빠른 배송, 세금계산서 발행 가능 |

### 상품 등록 시 입력 항목 안내

| 항목 | 설명 |
|------|------|
| 소싱처 종류 | **해외** 또는 **국내** 선택 |
| 소싱처 이름 | 예: 타오바오, 도매꾹, 사입삼촌 |
| 소싱 URL | 해당 상품 페이지 링크 |
| 최저가/도매가 | 소싱처에서 제공하는 최저 도매단가 |
| 최소주문수량 | 최소 발주 가능 수량 (MOQ) |
| 관부가세 | 해외 상품 수입 시 발생하는 관세·부가세 여부 |

### 반품/실거래 관리 체크리스트

상품 목록의 **반품** 열을 클릭하면 아래 항목을 체크할 수 있습니다:

| 항목 | 설명 |
|------|------|
| ⚠️ 반품 발생 | 해당 상품에 반품이 발생했음을 표시 |
| 💰 환불 완료 | 구매자에게 환불 처리 완료 |
| 📞 고객 연락 | 반품 관련 고객과의 연락 완료 |
| 📦 재고 반영 | 반품된 상품을 재고에 다시 반영 |
| 🚚 배송비 처리 | 반품 배송비 정산/처리 완료 |
| 🧾 세금계산서 발행 | 국내 도매 거래 시 세금계산서 수령 여부 |

## 💰 저가·소액 상품 운영 팁 (마진 최적화)

### 소액 상품 기준 및 전략

> 단가 **3만 원 이하** 소액·소량 상품 중심의 구매대행/도매 판매에 적합한 운영 방법입니다.

#### 1. 마진율 목표 설정

| 구분 | 권장 마진율 | 이유 |
|------|------------|------|
| 소액(~3만 원) | **30% 이상** | 절대 마진(원)이 작으므로 마진율을 높게 유지해야 수익 확보 가능 |
| 중가(3~10만 원) | **25~30%** | 광고비·CS 비용 감안 |
| 고가(10만 원+) | **20~25%** | 절대 마진이 크므로 마진율이 낮아도 OK |

#### 2. 묶음배송 / 단체배송으로 원가 절감

- **해외 소싱(예: 타오바오)** — 여러 상품을 한 번에 묶어서 배송하면 개별 배송비보다 저렴  
  → 동일 공급업체 상품을 여러 종 한꺼번에 주문 → 합산 배송비로 단가 낮추기
- **국내 도매** — 최소주문수량(MOQ) 이상 묶음 발주 시 도매가 적용  
  → 소량이라도 도매꾹·오너클랜 등에서 **정해진 MOQ 확인 후 발주**

#### 3. 소액 상품 소싱 체크리스트

```
✅ 경쟁사 1페이지 확인 (로켓배송 비중 < 70%)
✅ 쿠팡·네이버 자동완성으로 수요 키워드 확인 (무료)
✅ 소싱처 최저가(도매가) vs. 판매가 마진율 30% 이상
✅ 최소주문수량(MOQ) 확인 — 처음엔 소량만 테스트
✅ 해외 소싱 시 관부가세 여부 확인 (150달러 이상 시 발생 가능)
✅ 국내 도매 시 세금계산서 수령 가능 여부 확인
✅ 반품/교환 정책 확인 (특히 해외 소싱)
```

#### 4. 마진 계산 예시 (소액 상품)

| 항목 | 금액 |
|------|------|
| 매입가 (도매가) | 5,000원 |
| 국내 배송비 | 3,000원 |
| 네이버 수수료 (3.5%) | 525원 (판매가 15,000원 기준) |
| 기타비용 | 500원 |
| **판매가** | **15,000원** |
| **마진(원)** | **5,975원** |
| **마진율** | **≈ 39.8%** |

> 💡 **팁**: 소액 상품은 절대 마진이 적으므로, 동일 상품을 **여러 채널(네이버+쿠팡 동시)** 에 등록해 판매량을 높이는 것이 핵심입니다.

#### 5. 국내 vs. 해외 소싱 비교

| 항목 | 국내 도매 | 해외 구매대행 |
|------|----------|-------------|
| 배송 속도 | 1~3일 | 2~4주 (중국 직배) |
| 재고 확보 | 쉬움 (즉시 재주문) | 리드타임 고려 필요 |
| 관부가세 | 없음 | 150달러 초과 시 발생 |
| 세금계산서 | 발행 가능 | 불가 (구매대행 특성상) |
| 가격 경쟁력 | 중간 | 낮은 원가 가능 |
| 반품·교환 | 용이 | 어렵고 시간 소요 |
| 추천 상품 | 생활용품, 문구, 소품 | 패션잡화, 소형 전자기기, 특이 아이템 |

## 🗂️ DB 구조 예시 (상품 데이터 필드)

각 상품(item)은 `data/items.json` 파일에 아래와 같은 구조로 저장됩니다:

```json
{
  "id": 1,
  "name": "미니 선풍기",
  "source_type": "국내",
  "source": "도매꾹",
  "url": "https://www.domeggook.com/...",
  "purchase_price": 5000,
  "wholesale_price": 4500,
  "min_order_qty": 10,
  "has_customs_tax": false,
  "naver_price": 14900,
  "coupang_price": 13900,
  "memo": "여름 시즌 인기 상품",
  "return_occurred": false,
  "return_checklist": {
    "refund_complete": false,
    "customer_contact": false,
    "stock_updated": false,
    "shipping_fee_handled": false,
    "tax_invoice_issued": false
  }
}
```

| 필드 | 설명 |
|------|------|
| `source_type` | 소싱처 종류: `"해외"` 또는 `"국내"` |
| `source` | 소싱처 이름 (예: 타오바오, 도매꾹) |
| `url` | 소싱 상품 URL |
| `purchase_price` | 실제 매입가 (원) |
| `wholesale_price` | 소싱처 최저가/도매가 (원, 참고용) |
| `min_order_qty` | 최소주문수량 (MOQ) |
| `has_customs_tax` | 관부가세 발생 여부 (`true`/`false`) |
| `tax_invoice_issued` | 세금계산서 발행 여부 (반품 체크리스트 내) |

## 🖥️ 방법 1 — 내 PC에서만 사용 (로컬 실행)

> PC를 켜야만 사용 가능합니다. 설치가 제일 간단합니다.

### 준비물

- Python 3.10 이상 설치 ([python.org](https://www.python.org/downloads/) 에서 무료 다운로드)  
  ⚠️ 설치 시 **"Add Python to PATH"** 반드시 체크!

### 설치 (최초 1회)

명령 프롬프트(cmd)를 열고 아래 명령어를 한 줄씩 복사·붙여넣기 하세요:

```
git clone https://github.com/nadahong99/salesassist.git
cd salesassist
python -m pip install -r requirements.txt
```

### 실행 (매번)

```
python run_app.py
```

실행하면 2초 뒤 브라우저에서 `http://localhost:8000` 이 자동으로 열립니다.  
종료하려면 이 창에서 **Ctrl+C** 를 누르세요.

## ☁️ 방법 2 — 클라우드에 배포 (PC 꺼져도 어디서나 사용)

> PC를 꺼도 항상 켜져 있습니다. 핸드폰·아이패드·다른 PC 등 어디서든 접속 가능합니다.  
> **Render** 라는 무료 클라우드 서비스를 이용합니다. 복잡한 설정 없이 버튼 몇 번으로 완료!

### 📋 배포 순서 (복잡한 거 없어요!)

**1단계 — GitHub 계정 만들기**

- [github.com](https://github.com) 에서 무료 가입

**2단계 — 이 저장소를 내 GitHub에 포크(복사)**

- 이 페이지 오른쪽 위 **Fork** 버튼 클릭 → "Create fork" 클릭

**3단계 — Render에 배포**

아래 버튼을 클릭하면 Render 사이트로 이동합니다:

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/nadahong99/salesassist)

> 버튼이 안 보이면 주소를 직접 복사하세요:  
> `https://render.com/deploy?repo=https://github.com/nadahong99/salesassist`

**4단계 — Render 가입 및 배포 완료**

1. Render 사이트에서 **GitHub 계정으로 가입/로그인**
2. "New Web Service" 또는 배포 화면이 뜨면 **"Deploy"** 버튼 클릭
3. 1~3분 기다리면 `https://salesassist-xxxx.onrender.com` 같은 주소가 생성됩니다
4. 그 주소를 북마크하거나 핸드폰에 저장하면 **어디서나 접속 가능!**

### 💡 Render 무료 플랜 안내

| 항목 | 내용 |
|------|------|
| 비용 | **무료** (가입만 하면 됨) |
| 접속 | PC·핸드폰·아이패드 등 인터넷만 되면 어디서나 |
| 주의 | 15분 동안 접속이 없으면 서버가 잠시 슬립 상태가 됨 (첫 접속 시 30초~1분 로딩) |
| 데이터 | 서버가 재시작되면 입력한 상품 데이터가 초기화될 수 있음 |

> 📝 데이터가 초기화되는 게 불편하다면, Render의 **Persistent Disk** 기능(유료)을 사용하거나 로컬 실행 방식을 선택하세요.

## 🔄 반품 체크리스트 사용법

### 반품 발생 등록 및 후속 조치 체크

상품 목록의 **반품** 열에서 각 상품의 반품 상태를 관리할 수 있습니다.

| 상태 | 표시 | 설명 |
|------|------|------|
| 반품 없음 | `반품 등록` (회색) | 아직 반품이 발생하지 않은 상태 |
| 반품 발생 | `⚠️ 반품 (N/5)` (빨강) | 반품이 발생했으며 후속 처리 진행 중 |

**체크리스트 열기**: 반품 열의 배지를 클릭하면 체크리스트 팝업이 열립니다.

```
┌──────────────────────────────────┐
│ 🔄 반품 체크리스트                │
│ 상품명: 블루투스 이어폰           │
│                                  │
│ [✓] ⚠️ 반품 발생                 │
│                                  │
│ 반품 처리 후속 조치:              │
│ [✓] 💰 환불 완료                 │
│ [ ] 📞 고객 연락                 │
│ [✓] 📦 재고 반영                 │
│ [ ] 🚚 배송비 처리               │
│ [ ] 🧾 세금계산서 발행           │
│                                  │
│         [취소]  [저장]           │
└──────────────────────────────────┘
```

### 체크리스트 항목 설명

| 항목 | 설명 |
|------|------|
| ⚠️ 반품 발생 | 해당 상품에 반품이 발생했음을 표시 |
| 💰 환불 완료 | 구매자에게 환불 처리 완료 |
| 📞 고객 연락 | 반품 관련 고객과의 연락 완료 |
| 📦 재고 반영 | 반품된 상품을 재고에 다시 반영 |
| 🚚 배송비 처리 | 반품 배송비 정산/처리 완료 |
| 🧾 세금계산서 발행 | 국내 도매 거래 시 세금계산서 수령/발행 확인 |

### 엑셀 내보내기 시 반품 정보

엑셀 다운로드 시 아래 열이 자동으로 포함됩니다:

- `반품 발생` — `Y` 또는 빈 칸
- `환불 완료`, `고객 연락`, `재고 반영`, `배송비 처리`, `세금계산서` — 각각 완료 시 `Y`

## 🔧 문제 해결 (Troubleshooting)

### ❓ `pip`이 "내부 또는 외부 명령이 아닙니다" 오류

```
python -m pip install -r requirements.txt
```

> `pip` 대신 `python -m pip` 를 사용하면 됩니다.  
> 그래도 안 되면 Python을 [python.org](https://www.python.org/downloads/) 에서 재설치하고 **"Add Python to PATH"** 를 체크하세요.

### ❓ 로컬에서 같은 Wi-Fi 내 다른 기기(핸드폰 등)로 접속하고 싶을 때

```
ipconfig
```

### 1. 어떤 상품·키워드를 고르면 좋을까? (저가·중소형 중심)

| 항목 | 권장 기준 | 이유 |
|------|-----------|------|
| 단가 | **5,000 ~ 30,000원** 사이 | 초기 리스크 낮고, 재고 부담 최소 |
| 카테고리 | 식품·출산·의류·화장품 **제외** | 인증·AS·사이즈 이슈로 반품률 높음 |
| 경쟁 구조 | 쿠팡 1페이지에 해외배송 셀러 있고 리뷰 30개↑ | "팔린다"는 검증된 시장 신호 |
| 로켓 배송 비중 | **70% 미만** 키워드 | 그 이상이면 쿠팡 자체와 직접 경쟁이라 불리 |
| 트렌드 | 최근 3개월 검색량 상승 중 | 구글 트렌드·네이버 쇼핑인사이트로 무료 확인 |

**🔑 핵심 원칙**  
- 너무 흔한 1,000원 잡화보다 "리오더 이력 있는 소품"이 훨씬 안정적  
- 객단가 10만 원 이상 상품은 경쟁이 약한 대신 초기 자금·CS 부담이 크므로 초보에겐 비추천  
- 키워드는 "상품명"이 아닌 **"고객이 검색하는 문제/상황"** 기준으로 잡기  
  - ❌ "무선 이어폰" → ✅ "운동할 때 귀에서 안 빠지는 이어폰"

### 2. 국내 도매 + 해외 소싱 동시 관리 전략

#### 소싱처별 강점 비교

| 구분 | 국내 도매 (도매꾹·오너클랜·사입삼촌 등) | 해외 소싱 (타오바오·1688 등) |
|------|------------------------------------------|------------------------------|
| 납기 | 1~3일 (즉시 재고 가능) | 7~20일 (배송대행 포함) |
| 가격 | 상대적으로 높음 | 상대적으로 낮음 |
| AS/반품 | 간편 | 복잡 (셀러와 직접 소통 필요) |
| 상품 다양성 | 적음 | 매우 많음 |
| 추천 상황 | 급한 재고 보충·시즌 초기 | 마진 확보·차별화 상품 발굴 |

#### 실전 운영 팁

- **동일 or 유사 상품**이 국내외 모두 있으면 salesassist에서 소싱처를 분리 등록해 마진을 비교  
- 국내 도매 상품: **[국산]** 표기, 해외 소싱: **[직구/구매대행]** 등으로 상품명 구분 → 혼용 실수 방지  
- 합배송 활용: 타오바오·1688 소액 상품 여러 개를 **배송대행지에서 묶음** 발송하면 개별 배송비 절감 (수수료 분산 효과)  
- 시즌 전략:  
  - 명절·입학·여름·겨울 시즌 1~2개월 전에 해외 소싱 먼저 진행 (납기 고려)  
  - 시즌 직전·도중엔 국내 도매로 빠른 보충

### 3. 무료 키워드·시장 조사법 (유료 툴 없이)

유료 소싱 프로그램이 없어도 아래 방법으로 충분히 조사할 수 있습니다.

#### 무료 조사 도구 목록

| 도구 | 활용법 | 링크 |
|------|--------|------|
| 구글 트렌드 | 키워드 시기별 검색량·급상승 트렌드·지역별 관심도 | [trends.google.co.kr](https://trends.google.co.kr/) |
| 네이버 쇼핑인사이트 | 카테고리별 인기 검색어·트렌드 변화 | [datalab.naver.com](https://datalab.naver.com/shoppingInsight/sCategory.naver) |
| 쿠팡 자동완성 | 검색창에 한 글자씩 입력 → 실제 고객 검색어 확인 | 쿠팡 앱·웹 |
| 쿠팡 Wing 판매 데이터 | 내 상품의 노출·클릭·구매전환 지표 | [wing.coupang.com](https://wing.coupang.com/) |
| 스마트스토어 키워드 도구 | 키워드별 클릭률·쇼핑 연관어 제안 | 스마트스토어센터 광고 탭 |
| 오픈마켓 베스트셀러 | 카테고리별 실시간·주간 베스트 확인 | 쿠팡·11번가·옥션 베스트 탭 |

#### 실전 조사 순서 (엑셀 활용)

```
1. 구글 트렌드·쿠팡 자동완성으로 후보 키워드 20~30개 발굴
2. 엑셀에 키워드, 쿠팡 1페이지 상품 수, 로켓비중%, 최저가, 최고가 기록
3. 리뷰 수·판매건수·입점일을 같이 메모 → 경쟁강도 점수화
4. 경쟁사 리뷰 최신순 100개 읽기 → 불만·요청 키워드 집계
5. 해당 불만을 해결할 수 있는 상품을 타오바오·도매꾹에서 검색
```

> 💡 **핵심 습관**: 매주 월요일 30분, 위 순서대로 신규 후보 5개씩 발굴하면 분기마다 60개 이상 상품 파이프라인 확보

### 4. 반품·환불·리스크 실사례 및 입점 전 준비 체크리스트

#### ⚠️ 반품률 높은 상품 유형 (초보에게 비추천)

- 유리·도자기 등 **파손 위험** 상품
- 사용설명서가 어렵거나 설치가 복잡한 전자제품
- 사이즈·색상 오차가 큰 의류·신발
- 식품·건강기능식품 (유통기한·냉장 이슈)
- 배터리 내장 제품 (항공 배송 제한)

#### 입점 전 필수 체크리스트

```
□ 마켓별 수수료율 확인 (쿠팡 로켓그로스, 네이버 스마트스토어, 11번가 등)
□ 배송비 산정: 택배사 계약단가 + 포장재비 포함해서 계산
□ 부가세 포함 원가 산출 (salesassist 비용 설정에 부가세율 입력)
□ 소싱처 최소주문수량(MOQ) 및 초도물량 결정
□ 반품 정책 초안 작성 (반품 배송비 부담 주체 명확히)
□ 사업자등록증·통신판매업 신고 완료 여부 확인
□ 상품별 인증 필요 여부 확인 (KC 인증, 수입 허가 등)
□ 해외 소싱 시: 관부가세 + 배송대행 수수료 + 환율 변동 여유분 포함 원가 산출
```

#### 실사례 기반 주의사항

- **광고비 과투입 위험**: 검증 안 된 상품에 광고비 쏟으면 손실 확정 → 처음엔 하루 1,000~3,000원 소액으로 2주 테스트 후 확장  
- **가격 덤핑 경쟁 주의**: 시장 최저가보다 무리하게 낮추면 마진이 사라짐 → salesassist 마진 계산기로 최저 판매가 사전 설정 후 절대 그 이하로 내리지 않기  
- **초기 계정 관리**: 신규 셀러는 판매량이 갑자기 튀면 계정 점검을 받을 수 있으니 초반엔 자연스러운 판매 흐름 유지

### 6. 광고·썸네일·최적화 실수 예방법

#### 광고 전략 — 단계별 접근

| 단계 | 방식 | 예산 | 목적 |
|------|------|------|------|
| 1단계 | 수동 성과형 광고 (핵심 키워드만) | 하루 1,000~3,000원 | 클릭률·전환율 데이터 수집 |
| 2단계 | 데이터 검증 후 키워드 추가·입찰가 조정 | 하루 5,000~10,000원 | ROAS(광고수익률) 확인 |
| 3단계 | ROAS 검증 완료 후 자동/최적화 광고 전환 | 단계적 확대 | 노출 범위 확장 |

> ⚠️ 광고비는 판매가가 아닌 **마진 금액 기준**으로 설정해야 손실을 막을 수 있습니다.  
> 예: 마진 5,000원짜리 상품에 광고비 6,000원 쓰면 팔릴수록 손해

#### 썸네일 제작 체크리스트

```
□ 경쟁사 1페이지 썸네일 색상 분석 후 대비되는 색 선택
□ 배경색 있는 이미지 (흰 배경만 쓰면 눈에 안 띔)
□ 상품 핵심 소구 문구 1~2줄 (굵고 짧게, 읽는 데 1초 이내)
□ 실물 크기감·사용 상황이 느껴지는 사진 사용
□ 1:1 비율, 최소 1000x1000px 이상
```

#### 상세페이지 구성 순서 (전환율 중심)

```
① 상단: 경쟁사와 비교 — "왜 이 상품인가" 한 줄 설득
② 문제/공감대 제시 — 고객의 불편·불만 언급
③ 솔루션 제시 — 해당 상품이 어떻게 해결하는지
④ 스펙·성능 — 수치·실측 데이터 포함 (신뢰 확보)
⑤ 실사용 후기·Q&A — 자주 묻는 질문 미리 답변
⑥ 하단: 배송 안내, 교환·환불 정책 명확히 표기
```

#### 흔한 실수 모음

| 실수 | 예방법 |
|------|--------|
| 상품명에 키워드 난발 | 브랜드·소구포인트·메인키워드·세부키워드 순으로만 구성 |
| 광고 없이 오픈 후 방치 | 신규 상품은 초기 7일간 소액 광고 필수 (노출 확보) |
| 원가 계산 누락 (배송비·수수료·광고비) | salesassist 비용 설정에 모든 고정비 입력 후 마진 확인 |
| 경쟁사 최저가 무조건 따라내리기 | 최저 판매가 기준선을 미리 산출해 두고 그 이하 금지 |
| 첫 발주 대량 사입 | 소량(10~30개) 테스트 → 검증 후 증량 |
| 썸네일·상세 업로드 후 방치 | 2~4주 데이터 보고 클릭률 낮으면 썸네일 교체 |
